## 1. 프로젝트 개요
---
### 한 줄 정의
데스크탑 플로팅 공부 시간 타이머

### 제품 목표
사용자의 학습/작업 집중력을 향상시키고, 귀여운 움직이는 이미지를 통해 동기를 부여하며, 학습 시간 통계 리포트로 성취감을 제공하여 지속적인 자기계발을 돕습니다.

### 배경
현대 사회는 디지털 기기 사용의 증가로 인해 집중력 저하와 동기 부여 유지의 어려움을 겪고 있습니다. 특히 학습이나 작업 시, 효과적인 시간 관리 도구가 필요하지만 기존 타이머는 단순 기능에 그쳐 동기 부여에 한계가 있습니다. 사용자는 눈에 띄지 않게 시간을 관리하고 동기 부여를 얻을 수 있는 시각적이고 인터랙티브한 도구를 필요로 합니다.

### 사용자 문제
학습 또는 작업 중 집중력 유지의 어려움, 동기 부여 상실, 학습 시간 관리에 대한 명확한 피드백 부족.

### 해결 방안
데스크탑 화면에 항상 노출되는 플로팅 타이머를 제공하여 시간 인지를 돕고, 귀여운 움직이는 이미지를 통해 시각적인 즐거움과 감성적 동기 부여를 제공합니다. 또한, 학습 시간 통계 리포트로 성과를 시각적으로 보여줌으로써 지속적인 동기 부여와 목표 설정을 유도합니다.

### 차별점
단순히 시간을 측정하는 것을 넘어, '귀여운 움직이는 이미지'를 통해 사용자에게 감성적 동기 부여를 제공하고, 학습/작업 시간 통계 리포트를 통해 주도적인 성취감과 목표 설정을 돕는다는 점에서 차별점을 가집니다. 기존 타이머 앱들이 기능적인 측면에 집중하는 것과 달리, 사용자의 감정적 만족과 지속적인 동기 부여에 초점을 맞춥니다.

### 타겟 사용자
자기계발에 관심이 많고 학습이나 특정 작업에 집중해야 하지만 동기 부여 유지에 어려움을 겪는 일반 대중. 특히, 귀여운 캐릭터나 시각적인 요소를 통해 동기 부여를 얻는 것을 선호하는 학생, 직장인, 프리랜서 등.

### 사용자 시나리오
사용자는 공부나 작업을 시작할 때 '플로팅 공부 시간 타이머'를 실행하고, 목표 시간을 설정합니다. 데스크탑 화면 한 켠에 항상 떠있는 귀여운 캐릭터가 움직이며 시간을 세는 것을 보며 지루함 없이 집중력을 유지합니다. 타이머가 종료되면 그날의 학습/작업 시간을 통계 리포트로 확인하고, 주간/월간 성과를 분석하며 다음 목표를 설정합니다.

### 핵심 지표
월간 활성 사용자 수(MAU) 증가, 일일 평균 사용 시간, 사용자당 세션 수, 통계 리포트 조회율, 사용자 피드백을 통한 앱 스토어 평점 (4.5점 이상).

### 리스크
운영체제별 호환성 및 성능 문제 발생 가능성, 너무 많은 시각적 요소가 오히려 집중을 방해할 가능성, 기존 타이머 앱들과의 차별점 부족으로 인한 사용자 유입 한계, 개인정보 보호 및 데이터 보안 문제.

### 카테고리
> 생산성 / 업무

### 사용자 역할
> User

### 디바이스 (기기)
> Windows, macOS, Linux

## 1. 플로팅 타이머 기능
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

**수용 기준:**
```
1. ○ 타이머는 다른 창 위에 항상 떠 있어야 한다.
2. ○ 사용자가 타이머의 위치와 크기를 자유롭게 조절할 수 있어야 한다.
3. ○ 타이머의 투명도 조절 기능을 제공해야 한다.
4. ○ 카운트다운 및 스톱워치 모드를 지원해야 한다.

## 2. 움직이는 동기 부여 이미지
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

타이머와 연동되어 사용자의 집중을 유도하고 시각적 즐거움과 동기 부여를 제공하는 귀여운 움직이는 이미지를 제공합니다.

**수용 기준:**
```
1. ○ 타이머 작동 중 이미지가 설정된 애니메이션에 따라 움직여야 한다.
2. ○ 다양한 이미지 또는 캐릭터 테마를 선택할 수 있도록 제공해야 한다.
3. ○ 이미지 움직임이 너무 과도하여 집중을 방해하지 않아야 한다.
4. ○ 사용자가 이미지 표시 여부를 설정할 수 있어야 한다.
```


### 2.1 애니메이션 재생(타이머 연동)
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Windows, macOS, Linux

타이머 상태(진행/일시정지/종료)에 따라 캐릭터 애니메이션이 자연스럽게 재생/정지/변경됩니다.


#### 2.1.1 상태별 애니메이션 매핑

진행/일시정지/종료 상태에 각각 다른 애니메이션(예: 걷기/졸기/환호)을 매핑합니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Windows, macOS, Linux


#### 2.1.2 프레임/CPU 사용 제한

배터리/성능을 고려해 프레임 제한 또는 저전력 모드를 제공하고 CPU 사용이 과도하지 않도록 합니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Windows, macOS, Linux


#### 2.1.3 타이머 종료 연출

타이머 종료 시 짧은 축하/완료 연출을 재생하며, 기본값은 ON입니다. 사용자는 설정에서 끌 수 있어야 합니다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Windows, macOS, Linux


### 2.2 캐릭터/테마 선택
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Windows, macOS, Linux

사용자가 여러 캐릭터/테마를 선택하고, 미리보기로 움직임을 확인할 수 있습니다.


#### 2.2.1 캐릭터 목록/검색

캐릭터 목록을 제공하고 이름/태그로 검색할 수 있습니다(선택).
> **중요도**: 🟢 낮음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Windows, macOS, Linux


#### 2.2.2 미리보기 재생

선택 화면에서 캐릭터 애니메이션을 미리보기로 확인할 수 있습니다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Windows, macOS, Linux


#### 2.2.3 즐겨찾기

자주 쓰는 캐릭터를 즐겨찾기에 추가해 빠르게 선택합니다.
> **중요도**: 🟢 낮음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Windows, macOS, Linux


### 2.3 방해 최소화 옵션
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Windows, macOS, Linux

집중을 방해하지 않도록 애니메이션 강도, 프레임, 표시 위치/크기, 표시 여부를 세밀하게 설정합니다.


#### 2.3.1 애니메이션 강도(움직임 정도)

움직임을 약/중/강 또는 슬라이더로 조절해 시선 끌림을 줄입니다. 기본값은 '중'으로 설정합니다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Windows, macOS, Linux


#### 2.3.2 표시 여부 토글

캐릭터 표시를 완전히 끌 수 있어야 하며, 꺼도 타이머는 정상 동작합니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Windows, macOS, Linux


#### 2.3.3 위치/크기 독립 설정

캐릭터 위치/크기를 타이머와 독립적으로 조절할 수 있습니다(겹치지 않게).
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Windows, macOS, Linux

## 3. 학습/작업 시간 통계 리포트
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

사용자의 학습/작업 시간을 기록하고, 일별, 주간, 월간, 전체 기간별 통계 리포트를 시각적으로 제공하여 성과를 한눈에 확인할 수 있도록 합니다.

**수용 기준:**
```
1. ○ 일별, 주별, 월별 학습/작업 시간 기록 및 조회가 가능해야 한다.
2. ○ 누적 학습/작업 시간을 그래프 형태로 시각적으로 제공해야 한다.
